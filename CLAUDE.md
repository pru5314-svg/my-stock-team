# 주식 리서치 자동화 프로젝트

## 프로젝트 목적
종목명 하나를 입력하면 애널리스트(서브에이전트)들이 협업해 리서치를 만들고,
최종 결과를 **PPTX 리포트**로 내보내는 자동 리서치 시스템.

## 워크플로우
```
종목명 입력
  → 데이터 수집 (시세·재무·뉴스)
  → 애널리스트 협업 분석 (기술 / 펀더멘털 / 뉴스·심리)   ← 병렬 가능
  → 리스크 종합 (risk-manager)                          ← 위 3개 결과 필요 (순차)
  → 리서치 취합·검수 (research/)
  → PPTX 리포트 생성 (reports/)
```

1. **종목 접수** — `config/`의 종목 목록 또는 직접 입력
2. **데이터 수집** — `src/data/`에서 시세·재무·뉴스 수집 → `data/`에 저장
3. **애널리스트 분석** — `.claude/agents/`의 서브에이전트들이 분석 (병렬 여부는 아래 오케스트레이션 규칙 참고)
4. **리스크 종합** — `risk-manager`가 위 3개 결과를 모아 핵심 리스크·모니터링 포인트 도출
5. **리서치 취합** — 결과를 `research/<종목>/`에 모아 사실관계·일관성 검수
6. **리포트 생성** — `templates/`의 레이아웃으로 `reports/<종목>.pptx` 출력

## 애널리스트(서브에이전트)
`.claude/agents/`에 정의된 4종. 각자 독립 데이터 소스를 가진다.

| 에이전트 | 역할 | 데이터 소스 | 의존성 |
|---------|------|------------|--------|
| `market-technical-analyst` | 주가·추세·거래 동향 | FinanceDataReader (키 불필요) | 없음 (독립) |
| `fundamental-analyst` | 재무·실적·공시 | DART (`.env`의 `DART_KEY`) | 없음 (독립) |
| `news-sentiment-analyst` | 뉴스·이슈·시장 심리 | Claude Code 웹서치 (키 불필요) | 없음 (독립) |
| `risk-manager` | 결과 종합·리스크 점검 | pykrx (키 불필요) + 위 3개 결과 | **위 3개 완료 후** |

공통 규칙: 매수/매도·목표가 단정 금지, 모든 수치에 출처·기준일 표기, 한국어 출력, 면책 문구 포함.

## 오케스트레이션 규칙
**상황에 맞게 병렬 처리한다 — 무조건 병렬이 아니다.** 다음 기준으로 판단한다.

- **병렬로 실행 (권장)**: 작업이 서로 **독립적이고 입력이 겹치지 않을 때**.
  - 예: `market-technical-analyst` · `fundamental-analyst` · `news-sentiment-analyst`는 데이터 소스가 달라 서로를 기다릴 필요가 없다 → **한 번의 메시지에서 동시에 호출**한다.
- **순차로 실행**: 한 작업의 출력이 다음 작업의 **입력일 때**, 또는 앞 단계 결과에 따라 분기할 때.
  - 예: `risk-manager`는 위 3개 결과가 모두 나와야 종합할 수 있다 → 반드시 **3개 완료 후** 호출.
  - 예: 데이터 수집 → 분석 → 리포트 생성처럼 단계 간 의존이 있는 흐름.
- **단일 실행**: 사용자가 한 가지 관점(예: "기술적 분석만")만 요청하면 해당 에이전트 하나만 호출한다. 불필요하게 전체를 돌리지 않는다.

판단 기준 한 줄: **"입력이 서로 의존하는가?"** → 아니오면 병렬, 예면 순차. 토큰·시간을 아끼되, 정확성(의존성)을 깨면서까지 병렬화하지 않는다.

## 작업 규칙
- **데이터로 말한다** — 근거 없는 추측 금지. 수치·출처를 명시한다.
- **편향 없이** — 매수/매도 한쪽으로 치우치지 않고 리스크를 함께 제시한다.
- **투자 권유 아님** — 모든 리포트에 면책 문구(투자 판단 책임은 본인) 필수.
- **출력은 한국어** — 리서치·리포트·커뮤니케이션 모두 한국어.
- **산출물 경로 준수** — 중간 리서치는 `research/`, 최종 리포트는 `reports/`.
